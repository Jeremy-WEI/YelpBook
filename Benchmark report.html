Apache bench result: We have benchmarked our webpage using Apache Bench. The benchmark send concurrent reuqests to the server (1000 requests in total, 10 requests each time concurrently), the average response rate of server is tested. 

The most important factor we look for are requests per second and transfer rate (KB/sec). Several factors that affects the efficiency has been revealed. First, the loading of blank page (renders layout page) gives 124.54 kb/sec, while loading of a page with a typical MySQL query gives slower speed (going down from 59.44 requests/sec to less than 20 requests/sec. This shows that by executing sql queries, the speed of responding is affected.


Meanwhile, the loading speed from MongoDB (DB server is on mongoLab.com) is very fast. The speed for retrieving 1000 image files from MongoLab is 1395.24 KB/sec, vs about 100KB/sec for loading the node.JS website. This indicates that MongoDB is 

not a performance bottleneck for our website.
